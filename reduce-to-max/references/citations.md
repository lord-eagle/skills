# Citations

Each adopted rule in `principles/*.md` references one of these sources by short tag.

## NN/g — Nielsen Norman Group
- "How Little Do Users Read?" (J. Nielsen, 2008): readers scan; visitors read ~20% of words on an average page.
- "Above the Fold" research: critical content must appear in the first viewport.
- "F-Shaped Pattern for Reading Web Content" (J. Nielsen, 2006).

## Refactoring UI — A. Wathan, S. Schoger
- Establish hierarchy with size and weight; do not pile both with color and decoration.
- Limit type scale; limit accent count; lean on neutrals.

## Don't Make Me Think — S. Krug
- Self-evident pages > self-explanatory pages > nothing.
- Get rid of half the words on each page; then get rid of half of what's left.

## Dieter Rams — Ten Principles
- "As little design as possible." Less, but better.

## WCAG 2.2
- 1.3.5 Identify Input Purpose — `autocomplete` attribute on common fields.
- 1.4.3 Contrast (Minimum) — 4.5:1 body, 3:1 large text.
- 2.2.2 Pause, Stop, Hide — auto-playing motion content.
- 2.3.3 Animation from Interactions — animation must be disable-able where reasonable.
- 3.3.1 Error Identification — errors are described in text adjacent to the field.
- 3.3.2 Labels or Instructions — visible labels for user input.

## Whitney Hess — UX principles
- Stay out of people's way; build on others' successes.
